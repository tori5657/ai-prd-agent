# 产品背景

---

## 核心定位

**产品名称**：eSignGlobal（对外品牌：eSign.AI）

**一句话价值主张**：为中企出海及国际公司提供合规、安全的国际电子签名服务，以 AI 能力驱动合同执行效率提升。

> 对外定位：AI Contract Execution Platform（AI 合同执行平台）
> 内部核心：国际电子签平台，聚焦跨境合规签署场景

**服务模式**：SaaS，To B 为主

**数据中心**：香港 / 新加坡 / 法兰克福（客户按需选择数据存储地）

---

## 合规与认证

| 认证/标准 | 说明 |
|-----------|------|
| ISO 27001 / ISO 27018 | 信息安全与隐私管理 |
| GDPR | 欧盟数据保护合规 |
| EU Trusted List | 欧盟可信服务列表认证 |
| ESIGN / UETA | 美国电子签名法合规 |

## 产品线

| 产品线 | 阶段 | 优先级 | 首要目标 |
|--------|------|--------|----------|
| eSignGlobal 主产品 | 成长期 | P0 | 扩大国际市场份额，提升签署流程完成率和客户留存 |

**版本（Pricing Tier）**：

| 版本 | 定价 | 主要面向 |
|------|------|----------|
| Trial | 免费 | 个人/小团队体验，功能受限 |
| Basic | $30/月（年付）| 中小企业，100份/月签署额度 |
| Professional | 联系销售 | 中大型企业，含数字签名、SSO、API 基础集成 |
| Advanced | 联系销售 | 大型企业，含多工作空间管理、自定义角色权限、高级 API |

> 差异化策略：不按席位收费（Unlimited User Seats），与 DocuSign 等竞品形成价格对比优势。

## 核心能力架构概览

| 模块 | 职责描述 |
|------|----------|
| 签署（Signing） | 核心签署流程，含信封创建、签署区配置、多方签署、流程状态管理 |
| 信封模版（Envelope Template） | 可复用的签署流程配置模板，用于快速发起标准化签署任务 |
| 文件模版（Document Template） | 可复用的文件内容模板，支持变量填充，减少重复文档准备工作 |
| SmartForm | 智能表单，用于收集签署前的结构化信息，与签署流程联动 |
| AI 中心（AI Center） | 合同智能分析（风险识别/合规检查）、NLP 摘要、多语言翻译、自动起草 |
| 认证（Authentication/iAM） | 签署方身份核验，含密码验证、政府 ID 验证、iAM Smart（香港）、Singpass（新加坡）等 |
| TSP 对接（Trust Service Provider） | 对接可信服务提供商，提供 AES（高级电子签名）和 QES（合格电子签名）能力 |
| API | 开放平台，支持客户将签署能力集成到自有系统；分基础 API 和高级 API |
| 印章管理（Seal Management） | 企业印章和个人签名的上传、管理与授权，供签署时调用 |
| 用户管理（User Management） | 工作空间内的成员管理、角色与权限配置 |
| 登录（Auth/Login） | 账号体系、SSO（企业单点登录）、多工作空间切换 |

## 关键业务术语对照

| 术语 | 类型 | 定义 | 约束/备注 |
|------|------|------|-----------|
| 信封（Envelope） | 产品概念 | 一个完整的签署流程单元，包含文件、签署方配置和流程状态 | 签署任务以信封为粒度发起、追踪和管理 |
| 工作空间（Workspace） | 产品概念 | 类似企业空间，一个账号可加入多个工作空间，数据和权限在工作空间内隔离 | 工作空间是所有版本的基础概念；Advanced 版本新增「多工作空间管理」能力（同一账号跨工作空间切换） |
| 数据中心（Data Center） | 产品概念 | 客户数据的物理存储地，目前支持香港、新加坡、法兰克福 | 客户在注册/开通时选择，数据不跨区域流动 |
| TSP（Trust Service Provider） | 产品概念 | 可信服务提供商，eSignGlobal 通过对接 TSP 提供符合法律效力的高阶签名能力 | 支持 AES 和 QES 两种签名等级 |
| AES（Advanced Electronic Signature） | 枚举值 | 高级电子签名，具备更强身份绑定和不可否认性，满足欧盟 eIDAS 等法规要求 | 通过 TSP 对接实现 |
| QES（Qualified Electronic Signature） | 枚举值 | 合格电子签名，欧盟法律效力最高等级的电子签名，需使用合格设备和证书 | 通过 TSP 对接实现，适用于高合规要求场景 |
| 信封签署方 | 产品概念 | 被指定参与信封签署或审批操作的人，包含签署人（Signer）和审批人（Approver） | |
| 信封发起方 | 产品概念 | 创建并发起签署流程的人 | |
| 信封抄送方 | 产品概念 | 收到流程通知但无需操作的人 | |

> 更多术语随需求迭代持续补充至 `context/business-glossary.md`。
